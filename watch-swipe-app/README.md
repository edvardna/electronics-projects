# ⌚ Watch Swipe App

A swipe-based web application for exploring custom watch designs, combined with data collection to support design decisions.

---

## 🚀 Overview
The Watch Swipe App is a front-end project that introduces a **Tinder-style swipe interface** for browsing watch designs.

Beyond interaction, the application collects user preferences using **Firebase**, allowing data-driven insights into which designs users prefer. This enables more informed decisions when selecting which watches to develop or produce.

---

## 🧠 Features
- Swipe left / right interaction
- Dynamic watch display
- Clean, minimal UI
- Mobile-friendly layout
- Real-time data collection of user preferences

---

## 🛠 Technologies Used
- **HTML** → structure  
- **CSS** → styling and animations  
- **JavaScript** → swipe logic and interactivity  
- **Google Firebase** → storing swipe data and analyzing user preferences  

---

## 🎯 Purpose
The goal of this project was to:
- Explore **modern UI/UX patterns**
- Build an **interactive product browsing experience**
- Collect real user data to support **design decisions**
- Combine **frontend development with data-driven thinking**

---

## 📊 Data & Analysis
User interactions (likes/dislikes) are stored in Firebase, enabling:
- Tracking which designs are most popular  
- Identifying trends in user preferences  
- Supporting future product development decisions  

This transforms the project from a simple UI demo into a **data-informed product validation tool**.
